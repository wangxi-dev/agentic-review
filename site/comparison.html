Inspiration & why I rebuilt the wheel

Acknowledgement. The idea here is inspired by mdownreview — a polished native desktop app for browsing and reviewing AI-generated markdown and code. It nailed the core insight: humans need a fast, local loop to review what agents produce. agentic-review takes that same insight and rebuilds it around a different set of priorities.

So why rebuild the wheel instead of just using a great existing app?

Three reasons that matter to how I actually work. mdownreview is a beautifully engineered product; agentic-review deliberately optimizes for something else.

🪶

Lightweight — zero install, zero dependencies

The bridge is pure Python standard library and the review shell is static files. No build step, no package tree, no native toolchain, nothing to install. If a machine has Python 3 and git, it can run the entire review loop — versus installing, signing, and updating a native binary.

☁️

Portable — drop it in OneDrive, use it everywhere

Because it's just a folder of scripts and a static page, I keep it in OneDrive and every one of my devices picks it up automatically — no per-device install, no separate updater, no version drift. A native app means provisioning each machine on its own. Here the tool follows me; I don't re-install it.

🔁

Agentic-loop integration — and not tied to one agent

agentic-review isn't a separate app you alt-tab to — it's a skill any coding agent can drive (not just one vendor's). The agent launches a review, you comment, and take-feedback reads those comments straight back into the session so the agent iterates. The review is a turn in the working loop — coupling a standalone viewer can't give.

Where this is headed

💬 The agent comments back — and discusses

Today feedback flows one way: human → agent. The next step is making comments a two-way thread. When you leave a comment, the agent can reply inline — explaining a trade-off, pushing back with a reason, or asking a clarifying question — and you answer in the same thread. Review stops being a hand-off and becomes a discussion.

human comments → agent replies / discusses → human responds → agent revises

👀 A proactive loop — the agent watches for comments

Further out: instead of the human running take-feedback, the agent loops back on its own to proactively monitor the comment store. As soon as you leave a comment in the shell, a watching agent notices, picks it up, and starts acting on it — closing the loop without anyone re-invoking the skill. The review shell becomes a live channel between you and an agent that's always listening.
