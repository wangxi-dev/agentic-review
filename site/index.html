agentic-review

A local, human-in-the-loop review layer for AI-generated code.

AI agents produce more code than a human can comfortably review. agentic-review gives you a fast local loop to read, comment on, and feed changes back to the agent before it continues. The review UI talks only to a loopback server on your own machine — the one component with access to your files.

Read the setup guide

The review shell needs a running local bridge. If you launched with a token, use the ?token=… URL printed by agentic-review:launch.

How it works

Three pillars work together:

Skill

Teaches AI agents to launch a review, collect feedback, and clean up — via four plain-Python commands (stdlib only, cross-platform).

Shell

A static review UI (this site). Read diffs or browse every file, click a line or block to anchor a comment, and run code checks.

Local bridge

A loopback HTTP server on 127.0.0.1:8900 that exposes your local files to the shell and stores your comments.

The shell talks only to the local server, which is the only component with access to your files. An HTTPS GitHub Pages shell can call your http://localhost bridge via the browser loopback exception, or you can run everything same-origin straight from the bridge.

What you can do

  • Review the diff. By default the shell shows uncommitted changes (working tree vs HEAD); point the diff base at any ref to review committed history instead.
  • Browse all files. A Changed tab for the diff set and an All files tab that shows the whole repo as a collapsible tree (gitignored paths excluded).
  • Comment inline. Anchor a comment to a line, a range, or a Markdown block. Comments render inline in full, diff, and preview modes, and can be edited or deleted.
  • Rich renderers. Syntax-highlighted code, Markdown preview (with Mermaid diagrams), sandboxed HTML preview, and a collapsible JSON tree.
  • Run checks. Pluggable code checkers (lines-of-code and complexity built in) flag findings inline by severity. Drop your own CLI into .agentic-review/checkers/.
  • Propose a commit message. Stage a proposed commit message as a pseudo change so the human reviews it before you commit.

The skill commands

Run with your platform's Python 3 (python3, python, or py -3) — no third-party packages.

  • agentic-review:launch [comments-folder] — start the local server and print the shell URL.
  • agentic-review:precommit — stage a proposed commit message for review.
  • agentic-review:take-feedback — feed the stored comments back to the agent.
  • agentic-review:cleanup — shut down the server and remove temp comments.
